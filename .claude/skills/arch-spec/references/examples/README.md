# Examples — When to Read Which

Don't read all of them. Select based on the artifact type you're producing:

| What You're Writing | Read This First |
|--------------------|-----------------|
| overview / motif (母題) | `canonical-cycle-example.md` (see how it defines a one-liner + ten stages + MVP slice) |
| schema type | `schema-example.md` (see 7-layer structure + TypeScript + lifecycle state machine) |
| API type | `api-example.md` (see route mapping + resource graph + error envelope) |
| canonical slice / exemplar | `slice-example.md` (see concrete JSON + canonical story + cycle cadence) |
| demo path / validation path | `demo-path-example.md` (see phases + kill criteria + demo script) |

## Combination Reading

| Scenario | Read |
|----------|------|
| First time doing a spec stack | `canonical-cycle-example.md` + `schema-example.md` |
| Need grounding (too abstract) | `slice-example.md` + `demo-path-example.md` |
| Need to expose APIs | `api-example.md` + `schema-example.md` |
