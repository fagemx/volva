## 1. Terminal Intent

> **The user wants to make something that didn't exist before — with aesthetic intent, not utility intent.**

Core variables: 美學一致性, 完成度, 作品密度, recognizability, resonance

---

## 2. Space Builder Output

Space builder for expression regime produces **production formats**, not business vehicles:
- Short video essay
- Serialized visual micro-drama
- Storyboard-first world
- Interactive role/world format
- Image-sequence narrative pipeline

**Critical:** Candidates should NOT be driven by economic logic. Expression regime values resonance over monetizability.

---

## 3. Probe Form

Typical probe: "Can the user produce one complete piece in this format that they're proud of?"

Cheapest believable probe: a single production cycle with quality self-assessment.

---

## 4. Commit Threshold

- User can complete one piece in the format
- Style is recognizable (not generic)
- Production loop is sustainable (not a one-off heroic effort)

---

## 5. Forge Entry

CommitMemo for expression regime should specify:
- Production pipeline steps
- Asset templates or style references
- Quality bar (what "good enough" looks like)
- What NOT to build (e.g., don't build distribution before production is proven)
