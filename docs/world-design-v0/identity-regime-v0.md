---

## 1. Terminal Intent

> **The user wants to become a different version of themselves — not just do different things.**

Core variables: path viability, reversibility, identity fit, stability, long-term sustainability

---

## 2. Space Builder Output

Space builder for identity regime produces **staged transition paths**, not single projects:
- 兼職顧問路徑 (part-time consulting transition)
- 服務→產品過渡路徑 (service-to-product transition)
- Domain apprenticeship path
- Public-building path
- Operator transition path

**Critical:** Identity candidates are always staged — no single-step "become X" commitments.

---

## 3. Probe Form

Typical probe: "Can the user sustain step 1 of this path for 30 days without it being unsustainable?"

Cheapest believable probe: a 30-day trial of the first stage with sustainability assessment.

---

## 4. Commit Threshold

- First stage is sustainable (not burning out)
- Identity fit improves (the user feels more like "themselves")
- Path is reversible or low-cost to exit
- Next stage is visible (not blind commitment)

---

## 5. Forge Entry

CommitMemo for identity regime should specify:
- Current stage to build for (not the full path)
- Transition milestones
- Sustainability check schedule
- What NOT to build (e.g., don't build stage 3 infrastructure before stage 1 is proven)
