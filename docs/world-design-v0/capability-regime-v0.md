## 1. Terminal Intent

> **The user wants to get meaningfully better at something — not just know about it.**

Core variables: 能力增長, 熟練度, 可重現性, 速度, 品質門檻

---

## 2. Space Builder Output

Space builder for capability regime produces **learning carriers**, not opportunities:
- Practice loops (e.g., 12-day sprint)
- One-project curriculum
- Compare-and-rebuild workflow
- Daily output ladder
- Mentorless apprenticeship system

---

## 3. Probe Form

Typical probe: "Can the user demonstrably improve in 14 days with this practice structure?"

Cheapest believable probe: a 1-2 week practice loop with before/after quality comparison.

---

## 4. Commit Threshold

- Probe shows measurable skill gain
- User can sustain the practice cadence
- Quality bar is reachable, not asymptotic

---

## 5. Forge Entry

CommitMemo for capability regime should specify:
- Practice loop structure
- Progression milestones
- Quality benchmarks
- What NOT to build (e.g., don't build a full curriculum before testing one loop)
